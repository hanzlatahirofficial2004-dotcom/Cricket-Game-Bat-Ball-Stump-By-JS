# 🏏 Bat, Ball & Stump — The Ultimate Cricket Decision Game

A dynamic, interactive web-based logic game built with pure **HTML, CSS, and JavaScript**. Inspired by the classic Rock-Paper-Scissors logic, this project introduces a fun cricket-themed twist where users play against a randomized computer opponent.

---

## 🎮 How the Game Works
The logic follows a classic triangular strategy:
* **Bat beats Ball** (Bat swings and hits the ball!)
* **Ball beats Stump** (Ball hits the stumps and gets a wicket!)
* **Stump beats Bat** (Stumps catch the batsman out via run-out/stumping!)

## ✨ Features
* **Real-time DOM Manipulation:** Game results and choices are updated instantly on the screen without reloading.
* **Smart JavaScript Logic:** Utilizes pseudo-random number generation (`Math.random()`) to simulate an unpredictable computer opponent.
* **Persistent Score Tracking:** Keeps a live record of your Wins, Losses, and Ties using structured JavaScript objects and custom methods.
* **Clean & Dynamic UI:** Rounded image action buttons providing an intuitive gameplay experience.

## 🛠️ Tech Stack Used
* **Frontend Architecture:** HTML5 (Semantic structures)
* **Styling & Presentation:** CSS3 (Flex layouts, custom border radiuses, and responsive center alignments)
* **Game Engine:** Vanilla JavaScript (ES6 objects, functions, condition-handling, and DOM selectors)

---

## 📂 Code Highlights & Logic Structure
The core engine relies on a clean score-tracking object and a randomized selection function:

* **Score Tracker Object:** Manages game state internally and prints real-time stats via a dedicated display method.
* **Computer AI Engine:** Divides floating-point random numbers into three distinct probability brackets to select `Bat`, `Ball`, or `Stump` fairly.

## 🚀 How to Run Locally
1. Clone this repository or copy the code files.
2. Make sure you have an `images/` folder containing your game assets (`1.jpg`, `2.jpg`, and `3.jpg`).
3. Open the `index.html` file directly in any modern web browser.
4. Open the Developer Console (`F12`) if you want to inspect live state logs!
